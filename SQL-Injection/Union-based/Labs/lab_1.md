# Lab: UNION SQL Injection - Determining the Number of Columns

## Overview

В данной лабораторной работе рассматривается уязвимость **SQL Injection** в фильтре категорий товаров.

Приложение возвращает результаты SQL-запроса в HTTP-ответе, что позволяет использовать **UNION-based SQL Injection** для извлечения данных из других таблиц базы данных.

Первым этапом UNION-атаки является определение количества столбцов, которые возвращает исходный SQL-запрос. Этот шаг необходим для дальнейшего построения корректного UNION-запроса.

---

## Lab Objective

Определить количество столбцов, возвращаемых SQL-запросом, используя UNION SQL Injection.

Для успешного выполнения необходимо отправить SQL-запрос, который:

- использует оператор `UNION`;
- возвращает дополнительную строку;
- содержит правильное количество значений `NULL`.

---

## Ход работы

### Определение количества столбцов

Первым этапом UNION-based SQL Injection является определение количества столбцов, которые возвращает исходный SQL-запрос.

Для этого необходимо изменить параметр запроса и добавить оператор `UNION SELECT` с различным количеством значений `NULL`.

Если количество столбцов в нашем запросе совпадает с количеством столбцов оригинального запроса, база данных успешно выполнит объединение результатов.

---

### Проверка количества столбцов через UNION SELECT

Начинаем проверку с минимального количества столбцов:

```sql
' UNION SELECT NULL--
```

Если приложение возвращает ошибку, увеличиваем количество значений ```NULL```:

```sql
' UNION SELECT NULL,NULL--
```

Продолжаем увеличивать количество:

```sql
' UNION SELECT NULL,NULL,NULL--
```
